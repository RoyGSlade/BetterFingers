#### `process_recording_result` compatibility wrapper

A1.9 replaced the ~300-line inline body of `server.process_recording_result`
(`server.py:1440`) with 5 named stage closures (`transcribe`/`post_process`/
`no_audio_gate`/`rewrite`/`finalize`) run once through `DictationPipeline.run(...)`
(A1.6/A1.9, `server.py:1743-1756`). The function itself is now a compatibility wrapper:
admission gate (privacy-wipe check, single-flight `try_begin`, job creation,
`JOBS.transition(TRANSCRIBING)`), the `finally`-block safety net, and outcome translation
(completed/cancelled/failed) are unchanged code paths, re-verified by the board #16
read-only audit against `server.py:1440-1823`. Two cancellation sources are preserved via
`JobManagerCancellationBridge` (`backend/runtime/dependencies.py`), which ORs the real
`JOBS` registry with a directly-set `cancellation_event` — necessary because
`dictation_coordinator.cancel_active()` (privacy wipe) only sets the event and never
calls `JOBS.request_cancel`. One deliberate, documented behavior change: the `scratch
that` early-return previously left its job `FAILED` with a generic finally-block error
(no test asserted on it); it now completes normally via the same
`ctx.extra["_pipeline_stop"]=True` path the no-audio-gate stage already used. The board
#16 audit rated this change SAFE and adequately (not fully) tested, and recommended one
additional dedicated `scratch_that`-through-`process_recording_result` test as a
non-blocking follow-up — not yet added as of this entry.

### 2026-07-18 — I3.10 live-model integration harness

Added a live-model Message Rescue integration harness that can skip cleanly when no
local model is available and never reports that skip as a pass. New files only, no edits
to `server.py`, routes, or the renderer:

- `live_model_harness.py` — pure, injectable readiness classification
  (`discover_readiness`) and one-request preservation probe (`run_preservation_probe`) /
  orchestration (`run_harness`) logic. Reports are structural only (booleans, counts,
  category names, elapsed seconds) — never transcript or model content.
- `tools/live_model_harness.py` — CLI glue wiring the pure logic to the real
  `llm_engine.is_server_running`/`model_manager.get_model_file_status` readiness checks
  and `backend.services.rescue_llm_adapter.build_llm_call_fn` against the real sidecar.
  Never starts/stops/reloads the sidecar. Exit codes: `0` PASS, `2` UNAVAILABLE, `1`
  FAIL/CALL_FAILED — UNAVAILABLE is never exit `0`.
- `tests/test_live_model_harness.py` — 13 model-free tests against fakes (no network),
  covering readiness classification, the real-call-vs-safety-net distinction (a call that
  raises must report `CALL_FAILED` even though `rescue_message` still falls back to a
  preserving raw-transcript result), and that the probe report never contains transcript
  or context text.
- `tests/test_live_model_integration.py` — the live pytest test. Calls
  `pytest.skip(reason=...)` explicitly when `run_harness` reports `UNAVAILABLE`; otherwise
  asserts the real call was invoked without raising, the faithful variant is non-empty,
  every preservation category checked passed, and no context leak was detected.
